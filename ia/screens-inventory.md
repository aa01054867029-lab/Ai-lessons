# Инвентаризация экранов — ComplyFlow v1.0

## Список A: Экраны из PRD

| Имя экрана | Что на нём происходит |
|---|---|
| `Login` | Вход в платформу по email + паролю |
| `TwoFactorAuth` | Ввод TOTP-кода (2FA обязательна для CO и CEO) |
| `ForgotPassword` | Запрос ссылки для сброса пароля |
| `ResetPassword` | Установка нового пароля по ссылке из email |
| `DashboardCO` | Главный экран CO: счётчики новых клиентов, алертов (с разбивкой High/Medium/Low), pending approval requests |
| `AlertQueue` | Очередь всех алертов (KYC + транзакции + перечень РФМ): фильтры по severity/типу/дате/статусу, сортировка, bulk-действия (Approve/Reject) с чекбоксами |
| `AlertCard` | Карточка кейса: данные клиента, сработавшее правило, сумма/время, история транзакций за 30 дней, поле обоснования, кнопки Approve/Reject |
| `ClientsList` | Список всех клиентов с поиском и статусами KYC |
| `ClientCard` | Карточка клиента: документы, результат KYC (Валиден / Недействителен / Требует проверки), данные из перечня РФМ vs. данные клиента, история транзакций |
| `KYCUpload` | Загрузка фото паспорта (JPEG/PNG), индикатор обработки < 10 сек, отображение результата OCR |
| `ApprovalRequests` | Список входящих запросов на согласование для CO: кто запросил, что меняется, статус |
| `ApprovalRequestCard` | Карточка запроса: инициатор, текущее правило vs. предлагаемое изменение, обоснование PM, кнопки Approve/Reject с обязательным комментарием |
| `CreateApprovalRequest` | Форма PM: выбор правила для изменения, описание «До/После», обоснование |
| `EscalationCard` | Карточка эскалации к CEO: запрос PM + отказ CO + обоснование CO, кнопки «Поддержать блокировку» / «Override — принять риск» с обязательным текстовым обоснованием |
| `AuditTrail` | Неизменяемый журнал всех решений: дата, пользователь, роль, действие, объект, обоснование; фильтры по дате/пользователю/типу; экспорт JSON/CSV |
| `Settings` | Корень раздела настроек (навигация по подразделам) |
| `Profile` | Профиль пользователя: имя, email, фото |
| `Security` | Пароль, настройка 2FA, список активных сессий |
| `NotificationSettings` | Настройка каналов уведомлений (Email/Slack), триггеров (алерты High, эскалации, обновление перечня), режим Do Not Disturb |
| `TeamSettings` | Управление командой: список участников, роли (CO / CEO / Аудитор / Dev), отправка приглашений |
| `APIKeys` | Список API-ключей, создание нового ключа, отзыв; доступен роли Dev |

## Список B: Служебные экраны (в PRD не упоминаются, но без них продукт не соберётся)

| Имя экрана | Что на нём происходит |
|---|---|
| `InviteAccept` | Переход по инвайт-ссылке: проверка ссылки, выбор пароля, завершение регистрации |
| `OnboardingWelcome` | Первый вход нового CO в пустую платформу: краткий вводный тур (что настроить в первую очередь) |
| `EmptyState` | Пустое состояние очередей (нет алертов / клиентов / запросов) — подсказка, что делать дальше |
| `LoadingSkeleton` | Скелетон-загрузка для тяжёлых экранов (AlertQueue, AuditTrail) |
| `NotFound` | 404 — страница не найдена, кнопка вернуться на Dashboard |
| `ServerError` | 500 — критическая ошибка сервера, контакт поддержки |
| `Maintenance` | Заглушка на время плановых технических работ с ожидаемым временем восстановления |
