# UX-паттерны конкурентов

> Анализ на основе скриншотов Sumsub, данных deep research и отзывов пользователей (G2, Capterra, Reddit)

---

## 1. Рыночный стандарт — что делают все одинаково

Эти паттерны формируют **ожидания пользователей**. Отклонение от них = когнитивная нагрузка.

| Паттерн | Описание | Кто использует |
|---|---|---|
| **Sidebar + Content layout** | Левое боковое меню (навигация) + основная область контента справа. Фиксированный sidebar, scrollable content | Sumsub, ComplyAdvantage, Unit21, Salv, Ondato |
| **Карточка клиента / кейса** | При клике на алерт или клиента открывается детальная карточка: персональные данные, история транзакций, risk score, решение | Все 5 конкурентов |
| **Цветовая индикация риска** | 🔴 Красный = высокий риск, 🟡 Жёлтый = средний, 🟢 Зелёный = низкий. Используется повсеместно в таблицах, badges, графиках | Все 5 |
| **Фильтры в верхней панели** | Фильтрация по статусу, дате, типу, risk level — горизонтальная панель над таблицей/списком | Все 5 |
| **Статусные badges** | Визуальные метки: «Pending Review», «Approved», «Rejected», «Escalated» — на каждом элементе списка | Все 5 |
| **REST API документация** | Swagger/OpenAPI интерактивная документация с примерами запросов, sandbox для тестирования | Sumsub, ComplyAdvantage, Unit21 |

**→ Наш продукт должен реализовать все 6 паттернов как базовый UX-фреймворк.**

## 2. Точки расхождения — где решают одну задачу по-разному

| Задача | Подход A | Подход B | Подход C |
|---|---|---|---|
| **Настройка правил** | **Sumsub**: Visual branches (drag-and-drop дерево решений) | **Unit21**: No-code if/then builder (текстовые условия) | **ComplyAdvantage**: Настраиваемые параметры скрининга (слайдеры, threshold) |
| **Разбор алертов** | **Unit21**: Queue с risk scoring, приоритизацией, bulk-действиями, кастомными workflow-кнопками | **ComplyAdvantage**: Risk-based case routing, entity resolution | **Sumsub**: Карточка с AI-копилотом «Summy», который пишет summary |
| **Онбординг** | **Sumsub**: Полнофункциональный, но перегруженный — стартапы жалуются на кривую обучения | **Ondato**: Чистый и быстрый — фокус на скорости верификации | **ComplyAdvantage**: Минимальный — только скрининг, без KYC-формы |
| **Ценообразование** | **Sumsub**: Per-verification + скрытые микротранзакции | **ComplyAdvantage**: Entity-based + бесплатный tier (ComplyLaunch) | **Unit21**: Custom quote, средний $160K/год |
| **Управление архитектурой правил** | **Unit21**: Нет инструментов — при масштабировании «спагетти» | **Sumsub**: Визуальное дерево помогает, но без анализа конфликтов | Никто не решает проблему полностью |

## 3. Рекомендации — какие паттерны взять, у кого, почему

### Взять обязательно

| Паттерн | Взять у | Почему | Как адаптировать |
|---|---|---|---|
| **Alert queue с risk scoring** | Unit21 | Лучший UX для CO на 50-100 алертов/день: приоритизация, bulk, кастомные кнопки | Добавить IDF-взвешивание для кириллицы (снижение false positive с 90% до <15%) |
| **Visual workflow builder** | Sumsub | Интуитивный drag-and-drop, понятный без документации | Упростить до 3-5 типов узлов для MVP. Добавить ИИ-ассистент для обнаружения конфликтов правил |
| **AI case narrative** | Unit21 (Agentic AI) | Сокращение разбора с 14ч до 41мин/день, 100% precision | Реализовать LLM-суммаризатор с explainability (требование Кодекса этики ИИ ЦБ) |
| **ComplyLaunch модель** | ComplyAdvantage | Бесплатный tier → lock-in → конвертация при росте | Адаптировать: бесплатный план до 100 верификаций/мес |
| **Чистый UI верификации** | Ondato | Скорость верификации <60 сек = конверсия | Локализация: паспорт РФ, СНИЛС, ИНН |

### Создать с нуля (нет на рынке)

| Паттерн | Что делаем | Почему это новое |
|---|---|---|
| **Maker-Checker approval flow** | Diff-сравнение правил (До/После), эскалация к CEO, override с «риск принят», PDF для аудитора | Конкуренты работают с кейсами (алертами), но не с согласованием **системных изменений** политик |
| **Conditional routing по risk score** | Автоматическая маршрутизация: <100K руб. → менеджер, >1M руб. → CEO | Предотвращает организационный паралич (CEO не согласовывает каждый чих) |
| **Anti-spaghetti ИИ-ассистент** | При создании нового правила → ИИ анализирует конфликты с существующими и предупреждает | Главная боль Unit21: no-code правила при масштабировании превращаются в хаос |
| **BFF-прокси для РФМ** | JSON (REST) от стартапа → XML (SOAP) + ГОСТ → Росфинмониторинг | Все конкуренты = REST-only. Никто не решает проблему SOAP-шлюза |
| **Audit readiness dashboard** | Чеклист для CEO: «Готовы ли мы к проверке ЦБ?» — % выполнения требований | Нет ни у одного конкурента. Формирует привычку ежедневного захода в платформу |

### Не делать

| Паттерн | Пропустить | Почему |
|---|---|---|
| Mobile SDK (нативный) | MVP без мобильного SDK | Достаточно адаптивного веб + API. SDK = v2 |
| Collaborative intelligence (как Salv Bridge) | Не для MVP | Требует партнёрской сети между финтехами. Заложить в roadmap, но не сейчас |
| Графовая аналитика (как Unit21/Quantexa) | Для v3+ | Высокие затраты на ML + данные. Сначала — правильные правила, потом — графы |
