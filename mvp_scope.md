# MVP Scope

---

## Продукт в одном предложении

**ComplyFlow** — compliance-платформа для финтех-стартапов РФ, которая объединяет KYC-верификацию, AML-мониторинг по 115-ФЗ и approval workflow для разрешения конфликтов между продуктовой командой и комплаенсом в одном интерфейсе.

---

## Фичи MVP (Must Have)

| # | Фича | Что делает |
|---|---|---|
| F01 | **KYC-верификация** | Загрузка паспорта РФ → OCR → авто-проверка подлинности |
| F02 | **Перечень РФМ** | Скрининг каждого клиента по перечню экстремистов/террористов |
| F04 | **Проверка по МВД** | Валидация паспорта: действительность, утеря, просрочка |
| F07 | **Мониторинг транзакций** | Rule-based алерты: >600K, частота, аномалии. Предустановленные шаблоны 115-ФЗ |
| F08 | **Alert queue** | Очередь алертов с risk scoring, фильтрами, сортировкой по severity |
| F09 | **Bulk-действия** | Чекбоксы + массовое одобрение/отклонение однотипных алертов |
| F13 | **Approval workflow** | Запрос на изменение правила → CO рассматривает → Approve / Reject |
| F15 | **Эскалация + override** | CO отклоняет → запрос уходит CEO → Override с «риск принят» |
| F16 | **Audit trail** | Неизменяемый журнал: кто, когда, какое решение, обоснование |
| F26 | **RBAC** | 4 роли: CO (полный), CEO (dashboard + override), Аудитор (read-only), Dev (API) |
| F32 | **REST API + Webhooks** | POST /verify, GET /alerts, Webhook при изменении статуса |
| F46 | **Push-нотификации** | Email + Slack: эскалации, срочные алерты, обновления перечня |

## Что осознанно НЕ входит в MVP

### 1. Генератор ФЭС 1-ФМ / 2-ФМ (F19)
**Почему не сейчас:** Требует парсинга XML-справочников, интеграции с КриптоПро (ГОСТ), SOAP-шлюза к РФМ. Это 2-3 месяца инженерной работы. CO сейчас формирует ФЭС вручную — это боль, но не блокер для использования платформы. Появится в v1.1 как killer feature.

### 2. IDF-взвешивание + авто-dismiss (F10, F11)
**Почему не сейчас:** Основной скрининг (F02) работает и без IDF. CO будет получать false positives, но может обрабатывать их через bulk-действия (F09). IDF требует обучения модели на корпусе русских имён — это отдельная ML-задача. Добавляем в Sprint 2, до масштабирования.

### 3. CEO Dashboard (F24)
**Почему не сейчас:** CEO видит эскалации через уведомления (F46) и принимает решения через approval flow (F15). Отдельный dashboard с метриками — удобство, но не блокер. CEO заходит 1-2 раза/неделю. Добавляем в Sprint 2 как retention feature.

## Метрики успеха MVP

| Метрика | Цель | Как измеряем |
|---|---|---|
| **Time to first verification** | < 15 мин от регистрации | Время от sign up до первого POST /verify |
| **Алерты обработаны в день** | 50+ | Кол-во Approve/Reject/Escalate за рабочий день |
| **Среднее время на алерт** | < 3 мин | Время от открытия карточки до решения |
| **Approval requests/неделю** | ≥ 1 | Кол-во запросов через Maker-Checker (валидация core фичи) |
| **Retention CO (D7)** | > 60% | CO возвращается через неделю после первого визита |
| **NPS после 2 недель** | > 30 | Опрос CO: «Рекомендовали бы коллеге?» |
