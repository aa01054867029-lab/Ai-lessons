# Директива: Scope и приоритизация фич

## Задача
Собрать из всех артефактов проекта полный список фич, приоритизировать по MoSCoW и определить scope MVP.

## Контекст

Прочитай все файлы проекта:
- Бриф (brief.md)
- Конкурентный анализ (competitive_analysis_final.md)
- UX-аудит (audit/audit_report.md), если есть
- Персоны (personas.md)
- Итоги интервью (interview_*.md), если есть

## Инструкция

### Шаг 1 — Сбор фич

Пройди по всем файлам и собери полный список фич и требований. Источники:

- **Бриф** — описанные сценарии и требования
- **Инсайты из персон** — конкретные решения из блока 6 каждой персоны
- **Конкурентный анализ** — table stakes (есть у всех, нужно и нам) + дифференциаторы (есть у нас, нет у конкурентов)
- **UX-аудит** — паттерны, которые стоит перенять; проблемы, которых нужно избежать
- **Интервью** — edge cases и рекомендации из итогов

По каждой фиче укажи:
- Название
- Описание (1-2 предложения)
- Источник (откуда взяли)
- Для какой персоны критична

### Шаг 2 — Приоритизация по MoSCoW

Распредели фичи по четырём категориям:

**Must have** — без этого продукт не решает основную задачу primary-персоны. Запуск без этих фич бессмыслен.

**Should have** — важно, но запуститься можно и без. Добавляем сразу после MVP.

**Could have** — полезно, но не критично. Если есть время и ресурсы.

**Won't have (now)** — осознанно откладываем. Не в этой версии.

Критерии для Must have:
1. Фича напрямую связана с JTBD primary-персоны
2. Без неё основной сценарий использования невозможен
3. Это table stakes — есть у всех конкурентов, пользователь ожидает по умолчанию

### Шаг 3 — Scope MVP

На основе Must have сформулируй scope MVP:

1. **Одно предложение** — что это за продукт и что он делает в MVP
2. **Список фич MVP** — только Must have
3. **Основной сценарий** — путь primary-персоны от входа до результата (какие экраны/шаги)
4. **Что осознанно НЕ входит** — топ-3 фичи из Should have, которые будут в следующей версии, с объяснением, почему не сейчас

## Формат результата

Сохрани в три файла:

- `features_list.md` — полный список фич с источниками
- `prioritization.md` — MoSCoW-таблица
- `mvp_scope.md` — scope MVP: описание, список фич, основной сценарий, что не входит
