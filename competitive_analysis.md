## Краткий вывод по каждому конкуренту

### Sumsub
Ближайший аналог нашему продукту — all-in-one платформа, закрывающая KYC, AML и fraud в одном интерфейсе. Сильные стороны: мощный no-code workflow builder, быстрая верификация (<60 сек), SDK для мобильных приложений. Слабость: нет встроенного approval workflow для разрешения конфликта «скорость vs. безопасность», нет специфики РФ/115-ФЗ. UX ориентирован на массовую верификацию, а не на сложные compliance-процессы внутри стартапа.

### ComplyAdvantage
Золотой стандарт в AML risk intelligence — AI + NLP для обновления рисковых данных в реальном времени. No-code rule builder позволяет не-техническим пользователям управлять правилами. Ключевая слабость: **не делает KYC** (нет верификации документов), только скрининг. Это значит, что стартапу нужен второй инструмент для полного onboarding. Программа ComplyLaunch для стартапов — хороший ориентир для нашей бизнес-модели.

### Unit21
Лучший UX для compliance-офицеров: no-code alert queue с приоритизацией, кастомные workflow-кнопки, чеклисты для стандартизации расследований, shadow mode для тестирования правил. Латентность <250ms для realtime-решений. Слабость: фокус на US/EU, нет поддержки российской юрисдикции. Однако паттерны UI/UX — лучший ориентир для нашей queue-системы на 50-100 алертов/день.

### Salv
Уникальный игрок: помимо стандартного AML-стека, предлагает **Salv Bridge** — платформу для обмена данными о подозрительных лицах между банками и финтехами (GDPR-compliant). Это концептуально интересно, но для российского рынка пока неприменимо из-за регуляторных ограничений. Сильный акцент на collaborative intelligence и recovery fraud-средств.

### Ondato
Эталон скорости верификации: KYC за 30-60 секунд, точность 99.8%, 192 страны. Сильная сертификация (ISO 27001, eIDAS). UI хвалят за чистоту и интуитивность. Слабости: дороговат для маленьких стартапов, интеграция с legacy-системами может быть сложной, case management — базовый уровень. Хороший ориентир для UX нашего KYC-модуля.

## Общие выводы для нашего продукта

### 🎯 Где наша ниша

**Ни один из конкурентов не закрывает все наши потребности одновременно:**

1. **Approval workflow (скорость ↔ безопасность)** — это наше уникальное преимущество. Ни один конкурент не предлагает встроенный механизм разрешения конфликта между продуктовой командой и комплаенсом с эскалацией к CEO. Все существующие платформы предполагают, что compliance-правила задаёт один человек, а конфликт решается за пределами инструмента.

2. **Родная поддержка 115-ФЗ и Росфинмониторинга** — глобальные платформы не поддерживают российскую специфику из коробки. Проверка по перечню Росфинмониторинга, формы отчётности для ЦБ, требования к СНИЛС — всё это нужно строить самим или интегрировать отдельно.

3. **«Старт для стартапов» в РФ** — ComplyAdvantage предлагает ComplyLaunch для startups, но без поддержки РФ. На российском рынке нет аналогичного предложения.

### 💪 В чём можем быть сильнее

| Преимущество | Обоснование |
|---|---|
| **Workflow «продукт ↔ комплаенс»** | Единственный на рынке approval-процесс с эскалацией и audit trail |
| **115-ФЗ из коробки** | Росфинмониторинг, формы ЦБ, СНИЛС, паспорт РФ — без интеграций |
| **Единая платформа для РФ** | KYC + AML + отчётность + политики в одном интерфейсе (vs. 2-3 инструмента у конкурентов) |
| **Понятный onboarding для стартапов** | Не enterprise-монстр, а инструмент, который можно запустить за день |

| Паттерн | Откуда | Как применить |

| **Shadow mode / бэктестинг** | Unit21 | Тестировать новые правила на исторических данных до продакшена |
